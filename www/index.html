Password and username support random generating. random generating using a population and a length, separated by space.

Population:
p: ascii printable, l:lower letter, a:letter, u:upper letter, d:digits, h: hex digits, H:HEX digits
ld, ad, ud: l and d, a and d, u and d

Length:
Just integer, like 10, 20

Example: l 6, p 4

list filter

pattern, -pattern or [-]field:pattern separated in spaces

field = l|url, t|tag|tags, n|note|notes, u|user|username, U|USER|USERNAME(match exactly), User|Username(match only prefix)
pattern = just plain string, or regexp, if you know it

Example: u:ariel.wangyun l:gmail.com will give gmail items with username containing ariel.wangyun.
wangyun will give items having string wangyun
-t:email u:changsheng will give items not tagged by email and username containing changsheng

Concepts

Private and public key are RSA private and public keys, private key for decrypt, public key for encrypt. RSA keys can be generated by openssl.
Disk private/public keys are used for loading/storing file in crypted. Disk private should protected by a password.
Session private/public keys are used for reading/writting memory in crypted.
